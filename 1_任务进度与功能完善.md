### 阶段2：数据增强（难度⭐⭐ 中等，优先级🟡 中）

#### 2.1 AI 总结（DeepSeek API 集成）
**描述**：一键生成"最优时段建议"，自动总结表格
```
点击"📋 AI 总结" → 弹窗显示：
  📊 最优时段：3月7日 14:00-15:00（4人有空）
  ⚠️  冲突：张三 3月8日全天没空
  💡 建议：定在3月7日下午14-16点
```

**技术实现**：
- **后端**（server.py）：新增 `/api/session/<sid>/summary` 路由
  - 组织表格数据（参与者、有空/没空统计、最优时段计算）
  - 调用 DeepSeek API（通过 requests 库）
  - 返回 JSON `{summary: "..."}`
  
- **前端**：新增按钮和弹窗，调用上述 API

- **环境配置**：在 Railway 添加环境变量 `DEEPSEEK_API_KEY`

**工作量**：⭐⭐ 6-8 小时
**可行性**：✅ 90%（依赖 DeepSeek 服务可用性、API 费用低）

**成本分析**：
- DeepSeek 按 token 计费，单次总结 ~200 token，成本 ¥0.01
- 月活万级用户若 10% 用此功能，月成本 ~¥10

#### 2.2 Excel 导出
**描述**：一键下载表格为 Excel，方便离线统计或分享给不用小程序的人
```
表格 → "📥 导出 Excel" → 下载 meetup_聚餐安排.xlsx
  含：日期、时段、参与者、有空/没空状态
```

**技术实现**：
- **后端**：安装 `openpyxl` 库，新增 `/api/session/<sid>/export` 路由
  - 读取 session 数据
  - 生成 Excel 表格（行=时段，列=参与者）
  - 用颜色标记有空（绿）/ 没空（红）
  - 返回文件下载链接或直接 stream 返回

- **前端**：新增按钮，触发下载

**工作量**：⭐⭐ 4-6 小时
**可行性**：✅ 98%（openpyxl 库成熟，功能明确）
