# 群约小助手 - 分析报告

## 产品

**时间调查工具**：发起人创建日期范围（最多14天）和时间范围（如9:00-21:00），参与者点击/拖拽选择有空的时段，自动计算每个时段的空闲人数。

**核心设计**：
- 单页面Web应用，300KB以内
- 纯前端实现，无服务器
- 会话数据用Base64编码存放在URL Hash中
- 支持无限轮传递：A填完后生成新链接给B，B填完后链接自动包含A+B数据，B再分享给C...

```

## 转为小程序的技术方案

### 核心改动

**当前Flask架构**：本地服务器 + 文件系统
```
浏览器UI ← → Flask(IP:5000) ← → JSON文件(sessions/)
```

**小程序版本架构**：云端服务器 + 云数据库
```
小程序UI ← → 云函数 ← → 云数据库（Session ID索引）
                   ↓
        跨网络、全国用户可访问
```

**关键差异**：
- **部署**: Flask需要常驻进程 → 云开发完全托管
- **网络**: 同WiFi限制 → 全国范围可用
- **扩展**: 本地JSON → 云数据库，支持无限并发
- **成本**: ¥0 → ¥0（微信云开发免费额度足够）

### 具体工作量

| 任务 | 工作内容 | 时间 |
|------|---------|------|
| **后端迁移** | 将Flask代码改写为云函数（createSession/getSession/updateAvail），测试API | 12小时 |
| **前端适配** | WXML重写（复用HTML布局逻辑），事件绑定到云函数调用 | 25小时 |
| **数据转换** | JSON会话格式兼容云数据库，迁移脚本编写 | 8小时 |
| **集成测试** | 各机型测试、性能优化、并发写入测试、bug修复 | 40小时 |
| **发布上线** | 申请账号、资质认证、微信审核、灰度发布 | 14天（并行） |

**总工作量**：85小时代码 + 14天审核时间

### Flask数据格式到云数据库的迁移

**当前Flask JSON格式**（sessions/abc123.json）：
```json
{
  "id": "abc123",
  "name": "三月聚餐",
  "dateS": "2026-03-06",
  "dateE": "2026-03-09",
  "hourS": 18,
  "hourE": 22,
  "expectedNames": ["张三", "李四"],
  "participants": [
    {
      "name": "张三",
      "color": "#FF6B35",
      "avail": {
        "2026-03-06": [18, 19, 20],
        "2026-03-07": [19, 20, 21]
      }
    }
  ]
}
```

**云数据库格式**（完全兼容，只需添加_id和createdAt）：
```javascript
// 集合：sessions
{
  _id: "abc123",      // 云数据库自动生成或使用Flask的id
  id: "abc123",       // 保留原Flask字段
  name: "三月聚餐",
  dateS: "2026-03-06",
  dateE: "2026-03-09",
  hourS: 18,
  hourE: 22,
  expectedNames: ["张三", "李四"],
  participants: [/* 同上 */],
  createdAt: 1709754000000
}
```

**迁移步骤**：
1. 读取本地sessions/目录中的所有JSON文件
2. 逐个上传到云数据库sessions集合
3. 验证数据完整性（会话数、参与者数）
4. 云端保留所有历史数据
```

**核心云函数**（Node.js）：
```javascript
// 创建新会话
exports.createSession = async (event) => {
  const db = cloud.database()
  const sessionId = cloud.getWXContext().requestID
  await db.collection('sessions').add({
    data: { ...event, _id: sessionId }
  })
  return { code: 0, data: sessionId }
}

// 获取会话
exports.getSession = async (event) => {
  const db = cloud.database()
  const res = await db.collection('sessions')
    .where({ _id: event.sessionId })
    .get()
  return res.data[0] || null
}

// 更新参与者时段
exports.updateAvail = async (event) => {
  const db = cloud.database()
  const { sessionId, name, date, hours } = event
  await db.collection('sessions').doc(sessionId)
    .update({
      data: {
        participants: db.command.set(...)
      }
    })
}
```

## 小程序的付费功能设计

### 功能划分

**免费版功能**（保留全部）：
- 创建调查（14天上限）
- 参与调查
- 填写时段
- 查看汇总（仅人数）

**付费功能（¥49/年）**：
1. **热力图** - 用颜色深度显示最优时段（深绿=所有人都有空，浅灰=无人有空）
2. **Excel导出** - 下载参与者和时段对应的表格
3. **调查无期限** - 当前仅支持14天，付费可扩展到365天
4. **群组复用** - 保存常用的参与者列表，快速创建新调查

### 变现漏斗

```
100万日活用户
    ↓
2%激活率 = 2万活跃用户（月活）
    ↓
1%转化率 = 200付费用户（月新增）
    ↓
¥49 × 200 = ¥9800/月 = ¥117600/年
    ↓
扣除云成本（¥50/月）= ¥9750/月净收入
```

**这个目标现实吗？**
