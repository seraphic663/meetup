# 安全收口执行清单（A）

日期：2026-03-19

## 目标
- 完成 API Key 风险收口
- 防止未来再次提交密钥
- 明确历史泄露的应急流程

---

## 1) 即时动作（必须）

1. 在 DeepSeek 控制台轮换 `DEEPSEEK_API_KEY`
   - 生成新 Key
   - 立即失效旧 Key
2. 更新运行环境
   - 本地用户环境变量
   - 部署平台（Railway Variables）
3. 功能验证
   - 本地 `summary` 可用
   - 线上 `summary` 可用

验收：旧 Key 失效，新 Key 生效。

---

## 2) 仓库扫描（每次发布前）

执行：

- `python scripts/security_guard.py --workspace --history`

通过标准：
- 工作区扫描无命中
- Git 历史扫描无命中

---

## 3) 历史泄露治理（按风险执行）

若历史扫描命中真实密钥：

1. 先轮换 Key（必须先做）
2. 选择历史清理方案（`git filter-repo` 或 BFG）
3. 强制推送更新后的历史
4. 通知所有协作者重新克隆仓库

注意：历史清理会改写提交哈希，必须团队同步。

---

## 4) 持续防护

- 禁止在代码、脚本、文档中放真实密钥
- 示例值统一使用 `<YOUR_DEEPSEEK_API_KEY>`
- 报错日志中不打印密钥
- PR 合并前执行安全扫描脚本
